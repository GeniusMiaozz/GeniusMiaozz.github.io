30天自制操作系统-OSCodeFile学习笔记(1-7天内容)

2020-02-25 / 15 min read

关键概念

  • 一个操作系统其实就是许多二进制数组成的映像文件，一个二进制数可以理解为一个电信号。

  • 软盘的第一个扇区成为启动区，一个扇区有512字节。计算机从最初的一个扇区开始读盘，然后去检查这个扇区最后2个字节的内容，如果不是“55AA”,就会认为这张盘上没有所需的启动程序，报一个不能启动的错误。几乎所有的操作系统都把加载操作系统本身的程序放在启动区里。

  • GDT：全局段号记录表，用来保存每个段的表示信息，这些表示信息包括段的大小、段的起始地址、段的管理属性。系统可以通过分段的方式避免多个程序的内存使用范围重叠。

  • IDT：中断记录表，记录了0~255的中断号码与调用函数的对应关系，根据中断号码决定调用哪一个函数进行处理。其设定方法与GDT很相似。

  • GDTR/IDTR：48位寄存器，低16位保存段上限，高32位保存起始地址。

  • 在段的属性里设有一个Gbit标志位，当标志位为1的时候代表段上限的单位是1页而不是1字节。

  • 在整个操作系统中，全局描述符表GDT只有一张（一个CPU对应一个GDT）。GDT可以被放在内存的任何位置，但CPU必须知道GDT的入口，也就是基地址放在哪里，Intel的设计者门提供了一个寄存器GDTR用来存放GDT的入口地址，程序员将GDT设定在内存中某个位置之后，可以通过LGDT指令将GDT的入口地址装入此积存器，从此以后，CPU就根据此寄存器中的内容作为GDT的入口来访问GDT了。GDTR中存放的是GDT在内存中的基地址和其表长界限。

  • PIC：可编程中断控制器，将8个中断信号集合为一个中断信号的装置。监视输入管脚的8个中断信号，只要有一个中断信号进来，九江唯一的输出管脚信号标称ON，并通知给CPU。为了处理更多的外部设备，将中断信号设为了15个，通过主从两个PIC进行控制。

  • 如果CPU同意受理中断，会令PIC向CPU发送两个字节的数据。PIC在向发送数据时，送去的数据是“0xcd 0x??”两个字节。CPU会把传来的数据当做程序来执行，“0xcd”相当于“INT”，“0x??”相当于一个中断号，通过这种方法完成利用一根管脚选择执行某一号的中断。

  • 鼠标相对于计算机的历史属于新型设备，因此在最初设计师虽然在主板上做了鼠标用的电路，但只要不执行激活鼠标的指令，就不产生鼠标中断信号，即即使鼠标向CPU发送了数据CPU也不会接受。所以要先让鼠标控制电路有效，然后再让鼠标本身有效，才能让鼠标正常工作。

  • 如何令操作系统对鼠标与键盘这些外设的活动做出响应？

    1. 鼠标对应IRQ12的中断，键盘对应IRQ1的中断。

    2. 首先要对GDT、IDT初始化，这些是必备基础条件。

    3. 然后对PIC进行初始化，使CPU能够接受来自不同地方的中断信号。

    4. 随后完成鼠标和键盘在IDT中的注册，如果鼠标键盘进行操作发生了中断，CPU会自动调用中断函数进行处理。

    5. 因为鼠标控制电路包含在键盘控制电路里，初始化键盘控制电路的同时也完成了初始化鼠标控制电路。

    6. 如果要输出键盘、鼠标在发生中断时产生的数据，可以利用一个FIFO缓冲区来记录这些数据，然后在系统运行时不断地查看缓冲区如果有数据则输出，在处理中断时输出数据会大大降低系统运行效率。

naskfunc.nas

  • DAY_3：利用汇编完成实现暂停CPU功能的函数。

  • DAY_4：利用汇编完成部分函数，并与C函数链接。主要包含：

    1.实现不同字节大小的内存写入。

    2.中断标志的置位、复位。

    3.中断标志位的压栈保存、弹栈读取。

  • DAY_5：利用汇编完成实现对GDTR、IDTR寄存器加载数据的函数。

    GDTR与IDTR结构体基本一样。需要取第一个32位数据的低16位和第二个32位数据，合并作为一个48位数据存入寄存器。

    MOV AX,[ESP+6] 取第一个32位数据低16位数据地址
    MOV [ESP+6],AX 第一个32位数据高16位的地址存入其低16位数据
    LGDT [ESP+6] 第一个32位数高16位的地址开始的输入加载至寄存器

  • DAY_6：利用汇编完成中断处理完成之后的“返回”功能，C中的“return”相当于“RET”，此处需要“IRETD”的功能，故采用汇编实现，详见书P120。

    void putfont8(char *vram, int xsize, int x, int y, char c, char *font);

    其中vram是图形缓冲区的开始地址，xsize是分辨率的X，x是绘制图形最左侧的X坐标，y是绘制图形最顶部的Y坐标，c是绘制的颜色，font是绘制图形的数据信息。

    5.利用像素点阵的方法，通过对字符数组的遍历，依次绘制每一个字符的图形，从而完成对整个字符串各字符图形的绘制。

    7.利用像素点阵的方法，完成一个鼠标指针图形的绘制。主要过程为：

    首先利用init_mouse_cursor8(char *mouse, char bc)，将鼠标图形的像素点颜色信息存入mouse开头的地址中（bc为背景色）。

    然后利用void putblock8_8(char *vram, int vxsize, int pxsize,int pysize, int px0, int py0, char *buf, int bxsize)，vram和vxsize是关于VRAM的信息，pxsize和pysize是绘制图形的大小，px0和py0分别是图形左边的X坐标和图形右边的Y坐标，buf是待绘制图形的存放地址，bxsize是待绘制图像分辨率的X即每一行含有的像素数。

第一阶段(1-7天)成品(Version-harib04g)总结

  • 这一阶段主要完成了如何做出一个是“操作系统”的操作系统，也就是从零到有的转变。操作系统的启动过程，主要是通过启动区中的加载操作系统本身的程序来完成的，本项目中操作系统的存储介质为FAT12格式的软盘，软盘的第一个扇区是启动区，简单的启动区工作过程如下：

    1. 设定启动区内容的装载到的内存地址，必须按照规定设置为 0x07C00。

    2. 然后有一段标准FAT12格式软盘专用的代码，这一段按照要求对内存中写入数据，这也是执行程序前必不可少的。

    3. 启动区程序主体。首先对寄存器初始化，然后从软盘的第二个扇区开始读盘。存放第二个扇区开始若干个扇区的内存首地址为 0x8200（0x8000 ~ 0x81ff 这一段内存留给了启动区的复制。），倒是没有特殊理由就是因为这一块内存没人使用。

    4. 一次读一个扇区，首先读一个柱面、一个磁头的 18 个扇区，然后读一个柱面、两个磁头的 36 个扇区，最后读十个柱面、两个磁头的 360 个扇区。

    5. 在读扇区的过程中，辅以复盘纠错处理。如果读扇区时发生了失败，重新读这一个扇区；如果连续五次尝试读一个扇区均失败，则操作系统启动失败。

    6. 根据实验测试发现：一般想一个空软盘保存文件时，文件名会写在 0x002600 以后的地方，文件的内容会写在 0x004200 的地方。

    7. 由于从启动区开始，把磁盘上的内容装载到了内存 0x8000 号地址，所以磁盘 0x004200 的内容就位于内存 0x8000 + 0x004200 = 0xc200 号地址处了，在读盘结束之后直接跳转至该地址处执行。
